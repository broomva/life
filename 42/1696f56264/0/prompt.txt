# Plan: Publish aiOS crates to crates.io to fix `cargo install arcan`

## Context

`cargo install arcan` (and `cargo install --git`) fails because 7 dependency crates aren't on crates.io. Arcan's Cargo.toml already uses the correct Rust pattern (`path + version` dual-spec), but the `version` fallback can't resolve because the crates don't exist in the registry. The Lago crates are already published; only aiOS crates and one lago adapter are missing.

## What needs to happen

### Step 1: Prepare aiOS workspace metadata

**File: `aiOS/Cargo.toml`** — Add `repository` to `[workspace.package]`:
```toml
repository = "https://github.com/broomva/aiOS"
```

### Step 2: Add missing `description` fields to 5 aiOS crates

Each crate's `Cargo.toml` needs a `description`:
- `aiOS/crates/aios-runtime/Cargo.toml` — "Canonical Agent OS runtime engine"
- `aiOS/crates/aios-events/Cargo.toml` — "Event types and streaming for the Agent OS"
- `aiOS/crates/aios-policy/Cargo.toml` — "Policy engine and capability-based authorization for Agent OS"
- `aiOS/crates/aios-sandbox/Cargo.toml` — "Execution sandboxing for Agent OS tool invocation"
- `aiOS/crates/aios-tools/Cargo.toml` — "Tool registry and governance for Agent OS"

### Step 3: Add `version` to all aiOS internal path deps

Currently all aiOS internal deps use `path`-only (e.g., `aios-protocol = { path = "../aios-protocol" }`). For crates.io publishing, Cargo requires `version` alongside `path`:
- `aios-protocol = { path = "../aios-protocol", version = "0.1.0" }` (in 6 crates)
- `aios-policy = { path = "../aios-policy", version = "0.1.0" }` (in aios-tools, aios-kernel)
- `aios-sandbox = { path = "../aios-sandbox", version = "0.1.0" }` (in aios-tools)
- And all other cross-refs in aios-kernel

### Step 4: Publish aiOS crates in dependency order

```
1. aios-protocol        (no internal deps)
2. aios-runtime         (depends on aios-protocol)
3. aios-events          (depends on aios-protocol)
4. aios-policy          (depends on aios-protocol)
5. aios-sandbox         (depends on aios-protocol)
6. aios-tools           (depends on aios-protocol, aios-policy, aios-sandbox)
```

Publish with `cargo publish -p <crate>` from `aiOS/`, waiting ~30s between each for indexing.

### Step 5: Publish lago-aios-eventstore-adapter

**File: `lago/Cargo.toml`** already has `aios-protocol = { version = "0.1.0", path = "..." }` — once Step 4 is done, publish:
```

```

**`arcan/crates/arcand/Cargo.toml`** (lines 34-38 dev-deps):

**`arcan/Cargo.toml`** workspace deps — add the 4 missing crates:
