# Type System & Schemas

All foundational types are defined in `lago-core` and shared across the workspace. lago-core has zero external dependencies beyond `std`, `serde`, `ulid`, and `thiserror`.

## ID System

Lago uses **ULID** (Universally Unique Lexicographically Sortable Identifier) for all identifiers. ULIDs are 26-character base32 strings that encode a 48-bit millisecond timestamp + 80-bit random component, providing:

- Time-sortable ordering (newer IDs sort after older ones)
- Uniqueness without coordination
- Compact string representation

### Typed ID Wrappers

All IDs are newtype wrappers generated by the `typed_id!()` macro, providing type safety (you can't accidentally pass a `SessionId` where a `BranchId` is expected):

```rust
typed_id!(EventId);       // Unique event identifier
typed_id!(SessionId);     // Agent session identifier
typed_id!(BranchId);      // Branch within a session
typed_id!(RunId);         // Execution run grouping
typed_id!(SnapshotId);    // Snapshot identifier
typed_id!(ApprovalId);    // Approval gate identifier
```

Each generated type provides:
- `new()` — Generate a fresh ULID
- `from_string(s)` — Wrap an existing string
- `as_str()` — Borrow as `&str`
- Serde transparent serialization (serializes as plain string)
- `Display`, `Hash`, `Clone`, `PartialEq`, `Eq` traits

### Special Types

```rust
// Content-addressed blob identifier (SHA-256 hex digest, 64 chars)
pub struct BlobHash(String);

// Monotonic sequence number per branch
pub type SeqNo = u64;
```

## Event Model

### EventEnvelope

The `EventEnvelope` is the universal unit of state change. Every action in the system — a message sent, a file written, a tool invoked — is recorded as an event:

```rust
pub struct EventEnvelope {
    pub event_id: EventId,                  // Unique identifier (ULID)
    pub session_id: SessionId,              // Which agent session
    pub branch_id: BranchId,                // Which branch (default: "main")
    pub run_id: Option<RunId>,              // Optional execution grouping
    pub seq: SeqNo,                         // Monotonic sequence within branch
    pub timestamp: u64,                     // Microseconds since epoch
    pub parent_id: Option<EventId>,         // Causal parent link
    pub payload: EventPayload,              // Discriminated event type
    pub metadata: HashMap<String, String>,  // Extensible key-value metadata
}
```

### EventPayload

The payload is a tagged enum covering all agent side-effect types:

#### Session Lifecycle
| Variant | Fields | Description |
|---------|--------|-------------|
| `SessionCreated` | `name`, `config` | New agent session started |
| `SessionResumed` | `from_snapshot` | Session restored from snapshot |

#### LLM I/O
| Variant | Fields | Description |
|---------|--------|-------------|
| `Message` | `role`, `content`, `model?`, `token_usage?` | Complete LLM message |
| `MessageDelta` | `role`, `delta`, `index` | Streaming message chunk |

#### File Operations
| Variant | Fields | Description |
|---------|--------|-------------|
| `FileWrite` | `path`, `blob_hash`, `size_bytes`, `content_type?` | File created/updated |
| `FileDelete` | `path` | File removed |
| `FileRename` | `old_path`, `new_path` | File moved |

#### Tool Execution
| Variant | Fields | Description |
|---------|--------|-------------|
| `ToolInvoke` | `call_id`, `tool_name`, `arguments`, `category?` | Tool execution started |
| `ToolResult` | `call_id`, `tool_name`, `result`, `duration_ms`, `status` | Tool execution completed |

#### Approval Gate
| Variant | Fields | Description |
|---------|--------|-------------|
| `ApprovalRequested` | `approval_id`, `call_id`, `tool_name`, `arguments`, `risk` | Human approval needed |
| `ApprovalResolved` | `approval_id`, `decision`, `reason?` | Approval decision made |

#### Snapshots & Branches
| Variant | Fields | Description |
|---------|--------|-------------|
| `Snapshot` | `snapshot_id`, `snapshot_type`, `covers_through_seq`, `data_hash` | State checkpoint |
| `BranchCreated` | `new_branch_id`, `fork_point_seq`, `name` | Branch forked |
| `BranchMerged` | `source_branch_id`, `merge_seq` | Branch merged |

#### Policy & Extension
| Variant | Fields | Description |
|---------|--------|-------------|
| `PolicyEvaluated` | `tool_name`, `decision`, `rule_id?`, `explanation?` | Policy check recorded |
| `Custom` | `event_type`, `data` | User-defined event type |

## Core Traits

### Journal

The primary persistence interface. All implementations must be `Send + Sync` for use behind `Arc<dyn Journal>`:

```rust
pub trait Journal: Send + Sync {
    fn append(&self, event: EventEnvelope) -> BoxFuture<'_, LagoResult<SeqNo>>;
    fn append_batch(&self, events: Vec<EventEnvelope>) -> BoxFuture<'_, LagoResult<SeqNo>>;
    fn read(&self, query: EventQuery) -> BoxFuture<'_, LagoResult<Vec<EventEnvelope>>>;
    fn get_event(&self, event_id: &EventId) -> BoxFuture<'_, LagoResult<Option<EventEnvelope>>>;
    fn head_seq(&self, session_id: &SessionId, branch_id: &BranchId)
        -> BoxFuture<'_, LagoResult<SeqNo>>;
    fn stream(&self, session_id: SessionId, branch_id: BranchId, after_seq: SeqNo)
        -> BoxFuture<'_, LagoResult<EventStream>>;
    fn put_session(&self, session: Session) -> BoxFuture<'_, LagoResult<()>>;
    fn get_session(&self, session_id: &SessionId) -> BoxFuture<'_, LagoResult<Option<Session>>>;
    fn list_sessions(&self) -> BoxFuture<'_, LagoResult<Vec<Session>>>;
}
```

Uses `BoxFuture` (`Pin<Box<dyn Future<Output = T> + Send + '_>>`) instead of `async fn in trait` because the trait must be dyn-compatible for `Arc<dyn Journal>`.

### EventQuery

Builder pattern for filtering journal reads:

```rust
pub struct EventQuery {
    pub session_id: Option<SessionId>,
    pub branch_id: Option<BranchId>,
    pub after_seq: Option<SeqNo>,    // Exclusive lower bound
    pub before_seq: Option<SeqNo>,   // Exclusive upper bound
    pub limit: Option<usize>,
}

// Usage:
let query = EventQuery::new()
    .session(session_id)
    .branch(branch_id)
    .after(100)
    .limit(50);
```

### Projection

Event consumers that build derived state by replaying events:

```rust
pub trait Projection: Send + Sync {
    fn on_event(&mut self, event: &EventEnvelope) -> LagoResult<()>;
    fn on_replay_complete(&mut self) -> LagoResult<()> { Ok(()) }
    fn name(&self) -> &str;
}
```

### Mount

Virtual filesystem interface:

```rust
pub trait Mount: Send + Sync {
    fn read(&self, path: &str) -> BoxFuture<'_, LagoResult<Vec<u8>>>;
    fn write(&self, path: &str, data: Vec<u8>) -> BoxFuture<'_, LagoResult<()>>;
    fn delete(&self, path: &str) -> BoxFuture<'_, LagoResult<()>>;
    fn list(&self, prefix: &str) -> BoxFuture<'_, LagoResult<Vec<ManifestEntry>>>;
    fn exists(&self, path: &str) -> BoxFuture<'_, LagoResult<bool>>;
    fn stat(&self, path: &str) -> BoxFuture<'_, LagoResult<FileStat>>;
}
```

## Error Types

```rust
pub enum LagoError {
    Journal(String),
    Store(String),
    Serialization(serde_json::Error),   // #[from]
    SessionNotFound(String),
    BranchNotFound(String),
    EventNotFound(String),
    BlobNotFound(String),
    FileNotFound(String),
    SequenceConflict { expected: u64, actual: u64 },
    PolicyDenied(String),
    InvalidArgument(String),
    Io(std::io::Error),                 // #[from]
    Internal(String),
}

pub type LagoResult<T> = Result<T, LagoError>;
```

## Tool Span

Structured tracking for tool execution lifecycle:

```rust
pub struct ToolSpan {
    pub call_id: String,
    pub tool_name: String,
    pub arguments: serde_json::Value,
    pub category: Option<String>,
    pub status: Option<SpanStatus>,
    pub result: Option<serde_json::Value>,
    pub started_at: u64,
    pub ended_at: Option<u64>,
    pub duration_ms: Option<u64>,
}
```

A tool span begins with `ToolInvoke` and completes with `ToolResult`, linked by `call_id`. The `ToolSpan` struct reconstructs this lifecycle from events.
