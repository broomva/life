# Arcan-Autonomic Integration (R5 Phase 1)

## Context

Autonomic Phase 0 is complete — 5 crates, 69 tests, Lago wired, hysteresis active. The next step (R5) is wiring Arcan's agent loop to consult Autonomic's `/gating/{session_id}` endpoint before tool execution. This is the last missing piece to make the homeostasis controller actually influence agent behavior.

**Architecture**: Decorator pattern on `PolicyGatePort`. A new `AutonomicPolicyAdapter` wraps the existing `ArcanPolicyAdapter`, optionally consulting Autonomic over HTTP before delegating. Advisory semantics: if Autonomic is unreachable, Arcan continues with permissive defaults (fail-open).

## Step 1: Create `autonomic.rs` adapter in `arcan-aios-adapters`

**New file**: `arcan/crates/arcan-aios-adapters/src/autonomic.rs`

### Design

`AutonomicPolicyAdapter` implements `PolicyGatePort` as a decorator:

```rust
pub struct AutonomicPolicyAdapter {
    inner: Arc<dyn PolicyGatePort>,       // existing ArcanPolicyAdapter
    client: reqwest::Client,              // async HTTP client
    base_url: String,                     // e.g. "http://127.0.0.1:3002"
    economic_handle: EconomicGateHandle,  // shared state for economic gates
}

pub type EconomicGateHandle = Arc<tokio::sync::RwLock<Option<EconomicGates>>>;
```

### `evaluate()` flow

1. Call `GET {base_url}/gating/{session_id}` with 2s timeout
2. Parse response as `GatingResponse` (same struct used in `autonomic-api::router`)
3. Merge operational gates: compare `AutonomicGatingProfile.operational` with inner's decision → most restrictive wins
   - If Autonomic says `allow_side_effects: false` → Deny (regardless of inner)
   - If Autonomic says `max_tool_calls_per_tick < inner_limit` → use Autonomic's limit
4. Store `EconomicGates` in the shared handle for provider-layer hints
5. Fall through to inner on any HTTP error / timeout / parse failure (log warning)

### `set_policy()` flow

Delegate directly to `inner.set_policy()` (Autonomic doesn't own policy writes).

### Response type (for deserialization)

```rust
#[derive(Deserialize)]
struct GatingResponse {
    session_id: String,
    profile: AutonomicGatingProfile,
    last_event_seq: u64,
    last_event_ms: u64,
}
```

Note: `AutonomicGatingProfile` and `EconomicGates` are in `autonomic-core::gating`. We do NOT add a dependency on `autonomic-core` — instead, we duplicate the response types locally (3 small structs) to avoid coupling Arcan to Autonomic at the crate level. This keeps the advisory boundary clean.

## Step 4: Wire decorator in `main.rs`

**File**: `arcan/crates/arcan/src/main.rs` (lines 512-513)

Currently:
```rust
let policy_gate: Arc<dyn PolicyGatePort> =
    Arc::new(ArcanPolicyAdapter::new(aios_protocol::PolicySet::default()));
```

Change to:
```rust
let base_policy: Arc<dyn PolicyGatePort> =
    Arc::new(ArcanPolicyAdapter::new(aios_protocol::PolicySet::default()));

let (policy_gate, _economic_handle): (Arc<dyn PolicyGatePort>, Option<EconomicGateHandle>) =
    if let Some(url) = &resolved.autonomic_url {
        tracing::info!(url = %url, "Autonomic advisory enabled");
        let adapter = AutonomicPolicyAdapter::new(base_policy, url.clone());
        let handle = adapter.economic_handle();
        (Arc::new(adapter), Some(handle))
    } else {
        (base_policy, None)
    };
```

The `_economic_handle` is stored for future use by provider selection (R5 Phase 2 — not this PR).
