# R5 Phase 2: Close the Autonomic Feedback Loop

## Context

R5 Phase 1 is COMPLETE — `AutonomicPolicyAdapter` wired in Arcan, advisory gating works via HTTP. But the feedback loop is **open**: Arcan writes events to its journal, Autonomic has a separate journal with `last_event_seq: 0`. No rules fire because the projection is always at the default initial state.

**Problem**: The broadcast channel in `RedbJournal` is in-process only. Two separate processes sharing the same redb file can read/write concurrently (ACID), but live stream notifications won't cross process boundaries.

**Solution**: Embed Autonomic's pure controller (fold + rules) inside Arcan's process. The controller subscribes to the shared journal's in-process broadcast channel, maintains per-session projections, and evaluates rules locally. The standalone Autonomic daemon remains for multi-machine deployments and dashboards.

---

## Approach: Embedded Controller (Dual-Mode Adapter)

The `AutonomicPolicyAdapter` is refactored to support two modes:

- **Embedded** (default): Runs `autonomic-controller` in-process, subscribes to journal broadcast, microsecond-latency gating. No network.
- **Remote** (opt-in via `--autonomic-url`): Existing HTTP GET behavior, unchanged.

This preserves advisory semantics (failures fall through to inner gate) and the standalone daemon option.

## Step 3: Implement Embedded Controller

**New file**: `arcan/crates/arcan-aios-adapters/src/embedded_autonomic.rs`

### Design

```rust
pub type ProjectionMap = Arc<RwLock<HashMap<String, HomeostaticState>>>;

pub struct EmbeddedAutonomicController {
    projections: ProjectionMap,
    rules: Arc<RuleSet>,  // 6 default rules from autonomic-controller
    economic_handle: EconomicGateHandle,
}
```

**`spawn_journal_subscriber(journal: Arc<dyn Journal>)`**: Subscribes to journal's broadcast channel. On each event notification:
1. Read event from journal
2. `fold(state, event.kind, event.seq, event.timestamp)` → updated `HomeostaticState`
3. Store in projections map

**`evaluate_gating(session_id) -> Option<AutonomicGatingProfile>`**:
1. Read projection for session (returns None if no data yet)
2. `autonomic_controller::evaluate(state, rules)` → profile
3. Update `EconomicGateHandle` with economic gates

**Key properties**:
- Pure function calls only (fold + evaluate have zero I/O)
- Falls through to inner gate if no projection exists for session
- Microsecond-scale latency (in-memory read + pure computation)

## Step 6: Wire EconomicGateHandle to Provider Selection

**File**: `arcan/crates/arcan-aios-adapters/src/provider.rs`

The `ArcanProviderAdapter` (which wraps the LLM provider for the canonical runtime) accepts an optional `EconomicGateHandle`. Before each model call:

1. Read `EconomicGates` from handle
2. **Hibernate**: Block the call entirely (return error — runtime handles gracefully)
3. **Hustle**: Cap `max_tokens` to `min(requested, gates.max_tokens_next_turn)`
4. **Conserving**: Log advisory, cap tokens if set
5. **Sovereign**: No restrictions

This is advisory — if the handle is None or the read fails, the provider call proceeds normally.
