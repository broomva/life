we were reviewing this:

Completing Phase 2 (Self-Learning & Heartbeats) gives agents the ability to improve themselves over time. Here's what each piece unlocks:

2.2 Consolidation Job — Right now, learning capture (2.1) stores raw observations: "tool X failed with error Y", "user corrected Z". But they're just a pile of raw entries in the journal. Consolidation reads those entries, deduplicates them, clusters similar failures, and generates rule proposals — e.g., "always validate file paths before shell execution" or "prefer read_file over cat for large files". It turns noise into actionable intelligence.

2.3 Promotion Gate — Proposals from 2.2 are just suggestions. The promotion gate lets them become active policy rules through a governed pipeline — optionally requiring human approval, always with full rollback. This is the difference between "the agent noticed a pattern" and "the agent actually changed its behavior". It's the commit point for self-improvement.

2.5 Queue & Steering — This gives the agent loop proper concurrency semantics. Right now, messages are processed sequentially. With steering, you get:
collect — batch incoming messages while a run is active
steer — redirect the agent mid-run at safe tool boundaries
followup — queue a continuation after the current run
interrupt — safely preempt at tool boundaries

The big picture: After Phase 2, agents go from "stateless assistants that forget everything" to organisms that learn from mistakes, evolve their own rules, and handle concurrent demands gracefully. It's the self-learning score going from 2/10 to ~7/10.

This is the foundation the entire Life vision depends on — without it, agents are just fancy prompt chains. With it, they're genuinely self-improving systems.
