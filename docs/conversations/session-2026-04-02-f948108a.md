> **Research Summary**
> 
> **Noesis uses React/Ink** (a full TUI framework) — not raw ANSI. Key patterns:
> - **188 spinner verbs** ("Precipitating...", "Accomplishing...", etc.) — one picked randomly per turn
> - **Spinner glyphs**: `['·', '✢', '✳', '✶', '✻', '✽']` bounce-animated at 120ms/frame
> - **Status line**: `✻ Precipitating… (12m 3s · ↓ 2.0k tokens · thought for 17s)`
> - **Stall detection**: spinner color drifts toward red if no tokens for 3+ seconds
> - **Message queuing**: priority-based queue (`now > next > later`) with `useSyncExternalStore`
> 
> Since Arcan's shell is raw stdin/stdout (not Ink), we need to translate these into ANSI escape codes.
> 
> **Proposed Approaches**
> 
> **Approach A: ANSI Status Line (Recommended)**
> 
> A new `spinner.rs` module in the arcan binary crate. Uses a background thread that overwrites a single stderr line using `\r` + ANSI codes. Minimal dependencies — just `crossterm` for terminal width detection (already available via ratatui's crossterm).
> 
> ```
> arcan> What is 2+2?
> ✻ Precipitating… (3.2s)        ← overwrites in-place until first token
> Four.                           ← streaming output on stdout
> ✻ Done (4.1s · ↓ 847 tokens · $0.0012)
> arcan>                          ← prompt returns
> ```
> 
> For tools:
> ```
> [tool: bash] echo "hello"
>   ⠋ Running…                    ← spinner while executing
>   ✓ OK (0.3s)
> ```
> 
> Message queuing: stdin reader thread buffers lines while response is streaming, drains between turns.
> 
> **Approach B: Minimal Static Lines**
> 
> No animation — just print `Thinking...` before the provider call and `Done.` after. No elapsed time, no in-place updates. Simple but doesn't match the Claude Code experience.
>

> 
> Replace the REPL loop with a ratatui-based alternate screen (like `arcan chat`). Maximum fidelity but massive scope — essentially merging the TUI client into the shell.
