2. Key Technical Concepts:
   - **Git submodules with `--depth 1`**: CI uses shallow clones (`git submodule update --init --depth 1`) which can only fetch commits at current branch tips. If the parent repo records an older commit that's no longer a branch tip, the fetch fails with exit code 128 ("not our ref <sha>")
   - **Submodule stale refs**: Parent repo's recorded commit SHAs drift behind submodule remote main branch tips when submodule repos advance independently
   - **`.gitmodules` branch declarations**: Without `branch = main`, `--depth 1` doesn't know which branch to clone, making shallow fetches ambiguous
   - **concurrency: cancel-in-progress**: GitHub Actions concurrency setting that cancels in-progress runs when a new commit triggers the same workflow group
   - **Swatinem/rust-cache**: Rust build caching; if a run is cancelled before `Post Run`, the cache isn't saved, causing subsequent runs to rebuild from scratch
