# Phase 5: Connect Praxis FS Tools to Lago Event-Sourced Persistence

## Context

Praxis FS tools use raw `std::fs` calls. After every tool execution, `LakeFsObserver` in arcan's `main.rs` does a **post-hoc full-workspace scan**: replay all journal events ŌåÆ rebuild Manifest ŌåÆ snapshot entire workspace ŌåÆ diff ŌåÆ emit events. This is `O(workspace_size)` per tool call.

**Goal**: Lago should own the O(1) tracking path natively. A new `FsTracker` in lago-fs handles inline write tracking, producing the exact same `EventPayload::FileWrite` events that `ManifestProjection` already consumes. The O(n) `snapshot()` + `diff()` path is retained as a periodic reconciliation fallback ŌĆö not deleted.

**Architecture**:
