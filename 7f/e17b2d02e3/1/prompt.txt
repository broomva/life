# Plan: Extract Praxis — Canonical Tool Execution & Sandbox Engine

## Context

The Life project builds artificial life from AOS primitives. Tool execution (fs tools, sandbox, MCP bridge, hashline editing, skills) currently lives in `arcan-harness`, tightly coupled to `arcan-core::Tool` trait. This extraction creates **Praxis** ("action/practice") as a standalone project that any runtime can use, and unifies three competing sandbox abstractions.

**Current problems:**
1. `arcan-harness` tools implement `arcan_core::Tool` — unusable by other runtimes
2. Three sandbox abstractions exist (`aios-sandbox`, `arcan-harness::SandboxPolicy`, `lago-core::Sandbox`) with no shared types
3. `aios-protocol` has a minimal `ToolOutcome` (Success/Failure) while `arcan-core` has rich `ToolResult` with typed content blocks — the bridge loses information

**Outcome:** Praxis becomes the canonical tool execution engine. Any aiOS-compatible runtime depends on Praxis for tools. The `Tool` trait and rich result types move to `aios-protocol`.

## Architecture After Extraction

```
aios-protocol (canonical types: Tool trait, ToolDefinition, ToolResult, SandboxTier)
  │
  ├── Praxis (standalone tool execution engine)
  │     ├── praxis-core   — sandbox policy, workspace enforcement, command runner
  │     ├── praxis-tools  — ReadFile, WriteFile, EditFile, Glob, Grep, ListDir, Bash, Memory
  │     ├── praxis-mcp    — MCP server bridge (wraps external tools)
  │     └── praxis-skills — SKILL.md discovery and activation
  │
  ├── Arcan (runtime — consumes Praxis tools via ToolRegistry)
  │     ├── arcan-core     — re-exports Tool trait from aios-protocol
  │     ├── arcan-harness  — thin re-export shim (deprecated, removed later)
  │     └── arcan-aios-adapters — simplified bridge
  │
  └── Lago (persistence — SandboxTier imported from aios-protocol)
```

## Phased Implementation

### Phase 1: Extend aios-protocol with unified tool types

**Goal:** Move the rich tool types from `arcan-core::protocol` into `aios-protocol`.

**`aiOS/crates/aios-protocol/src/tool.rs`** — Add:
- `ToolAnnotations` (MCP-compatible: read_only, destructive, idempotent, open_world, requires_confirmation)
- `ToolDefinition` (name, description, input_schema, title, output_schema, annotations, category, tags, timeout_secs)
- `ToolContent` enum (Text/Image/Json — MCP-aligned)
- `ToolResult` (call_id, tool_name, output, content, is_error) — omit `state_patch` (Arcan-specific)
- `ToolContext` (run_id, session_id, iteration)
- `Tool` trait (sync: `definition() -> ToolDefinition`, `execute(&ToolCall, &ToolContext) -> Result<ToolResult, ToolError>`)
- `ToolRegistry` (BTreeMap<String, Arc<dyn Tool>>)
- `ToolError` enum (thiserror)

**`aiOS/crates/aios-protocol/src/sandbox.rs`** — New file:
- `SandboxTier` (None/Basic/Process/Container — from lago-core)
- `SandboxLimits` (max_runtime_secs, max_output_bytes, max_memory_mb)
- `NetworkPolicy` (Disabled/AllowAll/AllowList — from arcan-harness)
- `SandboxRunner` trait (async)

**`aiOS/crates/aios-protocol/Cargo.toml`** — Add `schemars`, `thiserror` deps.

**Backward compat:** `arcan-core::protocol` re-exports moved types as aliases. All downstream compiles unchanged.
