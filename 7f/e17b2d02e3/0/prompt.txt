### 2. AUTONOMIC-CONTROLLER CRATE

**Purpose**: Pure rule engine and projection reducer. No I/O dependencies.

#### Files:
- `Cargo.toml` - Depends on autonomic-core, aios-protocol, serde, tracing
- `src/lib.rs` - Module organization
- `src/engine.rs` - `evaluate()` function, decision merging
- `src/projection.rs` - `fold()` function for deterministic state reduction
- `src/economic_rules.rs` - SurvivalRule, SpendVelocityRule, BudgetExhaustionRule
- `src/cognitive_rules.rs` - ContextPressureRule, TokenExhaustionRule
- `src/operational_rules.rs` - ErrorStreakRule

#### Core Functions:

**`fold(state: HomeostaticState, kind: &EventKind, seq: u64, ts_ms: u64) -> HomeostaticState`**
- Pure deterministic projection reducer
- Consumes events and updates homeostatic state:
  - `RunFinished`: updates tokens, marks success, increments turns
  - `RunErrored`: increments error streak
  - `ToolCallCompleted`/`ToolCallFailed`: updates success/error counts
  - `StateEstimated`: updates mode and context pressure from aiOS
  - `BudgetUpdated`: updates token budget
  - `ModeChanged`: updates operating mode
  - `CircuitBreakerTripped`: updates error streak
  - `Custom` (autonomic events): processes cost/mode/credit changes
  - All other events: no-op
- Token cost calculation: (prompt_tokens × 3) + (completion_tokens × 15) micro-credits (default rates)
- Context pressure updated based on token consumption ratio

**`evaluate(state: &HomeostaticState, rules: &RuleSet) -> AutonomicGatingProfile`**
- Evaluates all rules against current state
- Merges decisions: most restrictive wins
  - Economic mode: highest severity (Hibernate > Hustle > Conserving > Sovereign)
  - Max tokens: minimum
  - Preferred model: cheapest
  - Restrictions: any true → true
  - Tool calls: minimum
- Special handling for Hibernate mode: locks down operational profile
- Special handling for Hustle mode: caps tool calls to 5, file mutations to 2

#### Rules:

**SurvivalRule**:
- Maps balance/burn ratio to EconomicMode
- Thresholds: Sovereign (≥2.0), Conserving (1.0-2.0), Hustle (0-1.0), Hibernate (≤0)
- No hysteresis in trait impl (hysteresis handled separately in engine if needed)

**SpendVelocityRule**:
- Configurable threshold (default: 500_000 micro-credits per 5 min)
- If cost_last_5min > threshold: suggest Budget model, cap to 2048 tokens

**BudgetExhaustionRule**:
- Configurable threshold (default: 20% remaining)
- If tokens_remaining_fraction < threshold: suggest Budget model, cap to 1024 tokens, restrict expensive tools

**ContextPressureRule**:
- Configurable threshold (default: 0.8)
- If context_pressure > threshold: suggest Standard model, cap to 2048 tokens

**TokenExhaustionRule**:
- Configurable threshold (default: 10% remaining, max 2 tool calls)
- If tokens_remaining_fraction < threshold: cap tool calls and tokens

**ErrorStreakRule**:
- Configurable error rate threshold (default: 30%) and min events (default: 5)
- If error_rate > threshold: restrict side effects, cap tool calls to 3
