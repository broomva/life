### Fixture format

Each file is a JSON array of Lago `EventEnvelope` objects (the canonical storage format):

```json
[
  {
    "event_id": "EVT-SC-001",
    "session_id": "GOLDEN-SIMPLE-CHAT",
    "branch_id": "main",
    "seq": 0,
    "timestamp": 1700000000000000,
    "payload": { "type": "SessionCreated", "name": "simple-chat", "config": {} },
    "metadata": {},
    "schema_version": 1
  }
]
```

Key: `seq: 0` because journal assigns monotonic sequences, overriding caller values.

const FIXTURES_DIR: &str = concat!(env!("CARGO_MANIFEST_DIR"), "/../../../conformance/fixtures");

2. Key Technical Concepts:
   - **Event-sourced architecture**: Lago stores all state as append-only events in a redb journal
   - **EventStorePort**: The canonical aios-protocol trait that `LagoAiosEventStoreAdapter` implements (append, read, head, subscribe)
   - **EventRecord vs EventEnvelope**: Protocol uses EventRecord (chrono timestamps), Lago uses EventEnvelope (u64 micros timestamps). The adapter converts between them via `protocol_bridge`
   - **EventPayload/EventKind**: `lago_core::event::EventPayload` is a type alias for `aios_protocol::EventKind` — the canonical event taxonomy
   - **Custom variant forward-compat**: Unknown event types deserialize to `Custom { event_type, data }`. After storage round-trip, they serialize as `{"type": "Custom", "event_type": "...", "data": {...}}`
   - **Journal sequence assignment**: The journal ignores caller-provided `seq` values and assigns monotonic sequences per session+branch
   - **redb**: Synchronous embedded DB — all operations wrapped in `spawn_blocking`
   - **BoxFuture**: Used for dyn-compatibility in the Journal trait (`Arc<dyn Journal>`)
   - **Submodule state**: The lago directory is a git submodule. CWD matters — running cargo from root vs lago directory resolves different workspace versions (v0.1.0 old vs v0.2.0 canonical)
   - **Rust 2024 Edition**: `gen` reserved, `set_var` unsafe, edition = "2024", rust-version = "1.85"
