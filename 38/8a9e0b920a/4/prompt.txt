# Autonomic: Homeostasis Controller for the Agent OS

## Context

The aiOS/Arcan/Lago stack has cognitive continuity (Arcan) and event persistence (Lago) but lacks a **regulation layer**. The CLAUDE.md already names it: *Autonomic — homeostasis controller and simulation kernel for agent stability regulation*. Today, homeostasis is hardcoded in `aios-runtime::apply_homeostasis_controllers()` with magic numbers and no economic awareness. Autonomic externalizes this into a proper rule engine with three co-equal pillars: operational, cognitive, and economic homeostasis.

The key insight from the Conway Research context is that **economics is a core concern from crate zero** — not a bolt-on. Agents need survival pressure: budget accountability, cost-aware model selection, and identity-based payments.

### Key Gap: GatingProfile Divergence

The existing `GatingProfile` (aios-protocol) has 6 **operational** fields. Autonomic needs **economic** fields: `economic_mode`, `max_tokens_next_turn`, `preferred_model`, `allow_expensive_tools`, `allow_replication`.

**Decision**: Autonomic defines `AutonomicGatingProfile` which embeds the canonical `GatingProfile` and adds economic extensions. This respects the kernel contract boundary. Arcan can consume the canonical portion immediately and adopt economic fields incrementally.

### Key Gap: Economic Events

No economic event types exist in `EventKind`.

**Decision**: Use `EventKind::Custom` with `event_type` prefixed `"autonomic."` (e.g., `"autonomic.CostCharged"`, `"autonomic.EconomicModeChanged"`). This is forward-compatible — the `custom_event_survives_round_trip` test in lago-aios-eventstore-adapter confirms Custom events round-trip through Lago. Promote to canonical EventKind variants once stabilized.

**Detailed steps:**

**Step 1: Projection event reducer** (`autonomic-controller/src/projection.rs`)
- Pure function: `fold(HomeostaticState, EventKind) → HomeostaticState`
- Handle: `RunFinished` (token cost), `ToolCallCompleted` (tool cost), `ToolCallFailed`/`RunErrored` (error tracking), `BudgetUpdated`, `StateEstimated`, `ModeChanged`, `Custom("autonomic.*")`
- Cost estimation: configurable per-model rates

**Step 2: Economic rules** (`autonomic-controller/src/economic_rules.rs`)
- `SurvivalRule`: balance / burn_rate → EconomicMode transitions with hysteresis
- `SpendVelocityRule`: cost_last_5_min > threshold → tighten gates
- `BudgetExhaustionRule`: cost_remaining < 20% → Conservative mode

**Step 3: Operational + cognitive rules**
- `ErrorStreakRule`: error_count/total > 30% → override to Verify mode
- `TokenExhaustionRule`: tokens_remaining < 10% → reduce max_tool_calls to 2
- `ContextPressureRule`: context_pressure > 0.8 → suggest model downgrade

**Step 4: Lago subscriber** (`autonomic-lago/src/subscriber.rs`)
- Subscribe to Lago journal via `Journal.stream(session_id, branch_id, 0)`
- Spawn per-session tailing tasks
- Feed events to controller engine, update shared projection state

**Step 5: HTTP API** (`autonomic-api/src/router.rs`)
- `GET /gating/{session_id}` → evaluate rules against projection → return `AutonomicGatingProfile`
- `GET /projection/{session_id}` → return raw `HomeostaticState` for debugging
- `GET /health` → daemon health check

**Step 6: Daemon** (`autonomicd/src/main.rs`)
- Config parsing (Lago URL, bind address, setpoints)
- tokio::main, graceful shutdown, tracing

## Communication Pattern

```
Arcan (tick_on_branch)
  │
  ├─ 1. Appends events to Lago journal (existing behavior, unchanged)
  │
  ├─ 2. GET /gating/{session_id} ──→ autonomicd (500ms timeout)
  │        AutonomicGatingProfile ←──────────────┘
  │
  └─ 3. Gates next action on profile (mode, model, tool permissions)

autonomicd
  │
  ├─ Subscribes to Lago SSE/journal stream (via autonomic-lago)
  ├─ Maintains per-session HomeostaticState projections
  ├─ Evaluates rule engine with hysteresis
  └─ Writes AutonomicDecision events back to Lago
```

**Pull model**: Arcan consults Autonomic before each tick. No Arcan architectural change. Failures are non-fatal (advisory).
